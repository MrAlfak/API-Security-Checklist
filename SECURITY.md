# Security Policy

## Reporting a Vulnerability

We take the security of this project seriously. If you discover a security vulnerability within this repository or have suggestions for security improvements, please report it to us.

### How to Report

1. **DO NOT** open a public issue for security vulnerabilities
2. Instead, please use GitHub's Security Advisory feature:
   - Go to the "Security" tab of this repository
   - Click "Report a vulnerability"
   - Fill in the details

Alternatively, you can:
- Open a private security advisory on GitHub
- Contact the maintainers directly through GitHub

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if you have one)
- Your contact information (optional)

### Response Timeline

- We will acknowledge your report within 48 hours
- We will provide a detailed response within 7 days
- We will work on a fix and keep you updated on progress
- Once fixed, we will publicly disclose the vulnerability (with credit to you, if desired)

### Scope

This security policy applies to:
- Security recommendations in the checklist
- Code examples provided in EXAMPLES.md
- Documentation and guidance

### Security Best Practices for Contributors

If you're contributing to this project:

1. **Review Security Content**: Ensure all security recommendations are accurate and up-to-date
2. **Verify Code Examples**: Test all code examples for security vulnerabilities
3. **Cite Sources**: Always provide references for security claims
4. **Stay Updated**: Keep track of latest security advisories and OWASP updates
5. **No Sensitive Data**: Never include real API keys, passwords, or sensitive data in examples

### Recognition

We appreciate security researchers and contributors who help keep this project secure. Contributors who report valid security issues will be:

- Credited in the repository (if desired)
- Mentioned in release notes
- Added to our contributors list

Thank you for helping keep API Security Checklist and its users safe!
